Option A — explicit line-height on message

Forces line boxes to a fixed height so emoji cannot inflate them. 1lh on icon/close = the same fixed height → always matches.

Option B — phantom text + align-items: baseline

A zero-width invisible span inside the icon gives it a text baseline. align-items: baseline on the row aligns all baselines together. Emoji doesn't change the text baseline — only the leading — so alignment is unaffected.

Option C — text-box: trim-both cap alphabetic

Trims leading space above/below text to cap-height and alphabetic baseline. The emoji's extra height falls outside the trimmed line box and has no layout effect. Chrome 123+ only.
