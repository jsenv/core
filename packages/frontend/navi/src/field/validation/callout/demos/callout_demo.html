Important: Callouts are designed to be displayed one at a time. In this demo, we show multiple callouts simultaneously for comparison purposes, but in practice only a single callout should be open to avoid overlapping issues.

Anchor Element Styling: Notice how each text element's border color changes when a callout is attached. This is achieved through the [data-callout] attribute and CSS custom properties that automatically reflect the callout's status color.

Positioning Control: Use positioning attributes on the anchor element:
  • data-callout-position="above" - Prefers callout to appear above the element (but will fallback to below if less than 60% would be visible above)
  • data-callout-position="below" - Prefers callout to appear below the element (but will fallback to above if less than 60% would be visible below)
  • data-callout-position-force="above" - Forces callout above regardless of available space
  • data-callout-position-force="below" - Forces callout below regardless of available space
Arrow Positioning: Use data-callout-arrow-x to control arrow placement. Notice how the arrows point inside the element (accounting for border widths) rather than at the outer edges:
  • data-callout-arrow-x="center" - Centers arrow horizontally on the element
  • data-callout-arrow-x="end" - Positions arrow at the right edge of the element
  • Default behavior positions arrow at the left edge of the element (inside the border)
The thick borders on these examples help visualize how the arrow targets the content area, not the border edges.

Unanchored Callout

This section demonstrates a callout without an anchor element. While this is primarily a robustness feature, it can be useful when dealing with elements that have no visual appearance (like input type="hidden", invisible elements, or when validation errors apply to the form as a whole).

Callout Centered in Viewport

Click the button below to open a callout without an anchor:

The callout will appear centered in the viewport without arrows.

Use Cases: This pattern can be helpful when:
  • Validating hidden form fields (<input type="hidden">)
  • Showing form-level validation messages
  • Handling elements with display: none or visibility: hidden
  • Displaying general alerts that don't relate to a specific visible element
Scroll Behavior: When centered in the viewport, the callout will not follow page scrolling and this is intentional. The callout is designed to display a message to the user at the moment openCallout is called. Following scroll would likely move the message further from the UI element that triggered it in the first place, reducing its contextual relevance.

When no anchor is provided, the callout centers itself in the viewport as a fallback.
