The problem

When block-level content (text, flex rows, etc.) wraps inside an element with a max-width, CSS "shrink-to-fit" sizing expands the element to fill all available space — leaving trailing whitespace on the right. The red tint makes the unused space visible.

How the width is computed

Range.getClientRects() returns one rect per text line. We find the widest — that becomes the new content width.
